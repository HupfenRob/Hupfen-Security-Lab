# Modernizing the Security Telemetry Pipeline

## Objective

Modernize the Suricata telemetry pipeline by replacing traditional syslog-style alert forwarding with structured EVE JSON telemetry to improve Splunk searching, detection engineering, and incident investigations.

---

- pfSense
- Suricata
- Splunk
- Kali Linux
- SSH
- EVE JSON
- Syslog

| SIEM | Splunk |
| Attack Workstation | Kali Linux |
| Telemetry Format | Suricata EVE JSON |
| Primary Goal | Improve network telemetry quality |

---

## Project Summary

This project focused on improving the quality of network telemetry available for detection engineering and incident investigation.

The existing environment already had a functioning centralized logging pipeline using pfSense, Suricata, and Splunk. Rather than replacing that architecture, the project modernized the telemetry being forwarded by configuring Suricata to generate structured EVE JSON telemetry.

This improved the quality of the data available in Splunk, reduced reliance on plain-text syslog parsing, and provided better context for future detections, alerts, and investigations.

## Implementation Highlights

### Original Telemetry Pipeline

Before modernizing the telemetry pipeline, Suricata events were written to the pfSense system log and forwarded to Splunk through remote syslog. While this approach provided basic visibility, it did not preserve the rich structured telemetry generated by Suricata's native EVE JSON output.

![Original Telemetry Pipeline](diagrams/original_telemetry_pipeline.png)

---

### Enabling Secure Shell Access

Secure Shell (SSH) was enabled on the pfSense firewall to allow secure remote administration from the management workstation. This provided command-line access for validating configuration changes, locating Suricata log files, and verifying telemetry generation without relying solely on the web interface.

![Enable Secure Shell](screenshots/enable_secure_shell.png)

---

### Validating Remote Administration

After enabling SSH, remote connectivity was tested from the Windows management workstation. Successful authentication confirmed that the firewall could be securely administered from the command line, providing an efficient method for validating configurations and troubleshooting the telemetry pipeline.

![Successful SSH Login](screenshots/successful_ssh_login.png)

---

### Configuring Suricata EVE JSON Output

Suricata was reconfigured to generate structured EVE JSON telemetry rather than relying solely on traditional syslog-style alerts. Enabling EVE JSON preserves detailed event fields, allowing Splunk to index richer network telemetry and providing a stronger foundation for future detection engineering and incident investigations.

![EVE JSON Output Settings](screenshots/eve_output_settings.png)

---

### Locating the EVE JSON Log

After enabling EVE JSON output, the pfSense file system was inspected to locate the active Suricata EVE JSON log. Confirming the log location verified that structured telemetry was being generated successfully and identified the source that would ultimately be forwarded through the logging pipeline.

![Locating the EVE JSON Log](screenshots/locating_eve_json_file.png)

---

### Validating Structured Telemetry

Viewing the contents of the EVE JSON log confirmed that Suricata was generating structured JSON events rather than traditional plain-text syslog messages. The additional fields provide significantly more context for Splunk searches, alerting, and future detection engineering efforts.

![Validating Structured Telemetry](screenshots/json_return.png)

---

### Generating Network Alerts

Test traffic was generated from the Kali Linux attack workstation to validate that Suricata continued detecting network activity after the telemetry pipeline was modernized. Successful alert generation confirmed that the updated configuration preserved detection capabilities while providing richer structured event data.

![Suricata Alerts Generated](screenshots/suricata_alerts_generated.png)

---

### Structured Events in Splunk

The structured EVE JSON telemetry was successfully forwarded through the existing logging pipeline and indexed by Splunk. Compared to traditional syslog alerts, the enriched event data provides significantly more context for searching, correlation, alerting, and future detection engineering activities.

![Structured Events in Splunk](screenshots/suricata_event_in_splunk.png)

---

### Modernized Telemetry Pipeline

With validation complete, the telemetry pipeline now forwards structured Suricata EVE JSON events to Splunk while preserving the existing network architecture. The result is richer security telemetry, improved searchability, and a stronger foundation for future detection engineering, threat hunting, and incident response projects.

![Modernized Telemetry Pipeline](diagrams/updated_pipeline_diagram.png)
