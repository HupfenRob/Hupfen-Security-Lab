# Mission 3 Screenshots

This directory contains screenshots documenting the major implementation steps completed during Mission 3.

The images are referenced throughout the mission README to illustrate the deployment of the Ubuntu Server, Linux system hardening, secure remote administration, audit configuration, logging validation, and file integrity monitoring.

## Contents

| Screenshot | Purpose |
|------------|---------|
| `ubuntu_configuration.png` | Configuring the Ubuntu Server hostname, user account, and initial system settings during installation. |
| `remote_ssh_administration.png` | Validating secure remote administration using OpenSSH from the management workstation. |
| `linux_system_validation.png` | Confirming system configuration, user privileges, and baseline operating system functionality. |
| `linux_journalctl_logging.png` | Reviewing persistent system and authentication logs using journald. |
| `auditd_configuration.png` | Verifying the Linux Audit Framework (auditd) service and configured audit rules. |
| `aide_initialization.png` | Initializing AIDE to establish a trusted file integrity monitoring baseline. |
